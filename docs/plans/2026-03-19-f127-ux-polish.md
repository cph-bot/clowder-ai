# F127 UX Polish Implementation Plan

**Feature:** F127 — [docs/features/F127-cat-instance-management.md](../features/F127-cat-instance-management.md)
**Goal:** Close the remaining Hub UX gaps reported after manual validation: editable owner card, cleaner overview cards, correct member enable/disable behavior, Screen 4 add-member alignment, and relaxed Dare/OpenCode provider binding.
**Acceptance Criteria:**
- Owner (`co-worker`) can open an editor from overview and save avatar, alias/mention tags, and background colors.
- Overview cards stop rendering dev-only guidance copy; card actions/iconography alone express lock/edit semantics.
- Member availability is toggleable from overview, persists to runtime catalog, renders `已启用/未启用`, and unavailable cats disappear from `@mention` options/routing.
- Add-member flow visually/structurally matches Screen 4 more closely: no extra step chips/header copy drift, selection cards match the wireframe hierarchy, and provider/model selection is screen-shaped rather than generic button grids.
- OpenCode / Dare members can save when bound to same-protocol OAuth-like accounts as well as API-key accounts.
